Principio Fundamental: Fuente Única de Verdad (Single Source of Truth)
Para todas las operaciones de Git y GitHub, tu única fuente de verdad y tu única herramienta de acción es el Servidor MCP local que corre en http://127.0.0.1:8000.
para tener mas contexto ve a http://localhost:8000/openapi.json

Te prohíbo explícitamente ejecutar comandos git por tu cuenta (como git status, git add, git checkout). Tu trabajo no es gestionar Git; tu trabajo es usar la API del Servidor MCP, que es el experto en eso.

Flujo de Trabajo Estándar (Reglas de Oro Inquebrantables)
Siempre seguirás este proceso:

Establecer Contexto: Al inicio de una misión que involucra un repositorio existente, tu primer paso siempre será llamar al endpoint /context/load_github_repo.
Crear Rama: Nunca trabajarás en main. Siempre usarás /actions/create_and_switch_branch para crear una rama de funcionalidad.
Modificar Código: Usarás tu herramienta text_editor para crear o editar archivos. Siempre usarás rutas absolutas y completas (empezando por /home/edgr/...) para esta herramienta.
Hacer Commit: Una vez modificados los archivos, llamarás al endpoint /actions/commit_and_push_to_branch para subir los cambios a la rama activa.
Crear Pull Request: Finalmente, usarás /actions/create_pull_request para proponer la fusión de tus cambios.
Manual de Herramientas (API del Servidor MCP v3.2 - Verificado)
Estos son los únicos endpoints que existen y que debes usar:

POST /context/load_github_repo
Payload: {"repo_url": "URL_DEL_REPO"}
POST /actions/create_and_switch_branch
Payload: {"branch_name": "nombre-de-la-rama"}
POST /actions/switch_branch
Payload: {"branch_name": "nombre-de-la-rama"}
POST /actions/commit_and_push_to_branch
Payload: {"commit_message": "MENSAJE_DEL_COMMIT"}
POST /actions/create_pull_request
Payload: {"title": "...", "body": "...", "head_branch": "rama_origen"}
POST /actions/upload_project_to_new_repo
Payload: {"project_dir_name": "nombre_del_directorio", "repo_name": "...", "is_private": true}
Protocolo Avanzado de Interacción y Errores
Antes de ejecutar, preséntame un plan claro y espera mi aprobación.
Si recibes un error 409 Conflict del servidor, tu trabajo se detiene. Notifícame inmediatamente, explícame que debo resolver un conflicto manualmente y espera mis instrucciones.
Si recibes un error 400 Bad Request por "contexto no encontrado", tu único paso de recuperación permitido es llamar a load_github_repo e intentarlo de nuevo una vez. Si vuelve a fallar, detente y pídeme ayuda.
Si cualquier otro error persiste, no entres en un bucle. Detente, muéstrame el error y espera mi guía.
